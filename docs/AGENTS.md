# Documentation Scope

Inherits the repository-root instructions. This file applies under `docs/`.

- Keep user, architecture, validation, and release evidence distinct; label planned versus runtime-capable behavior precisely.
- Update the owning plan/ADR when behavior, contracts, or validation policy changes materially.
- Evidence pages link to maintained commands/artifacts and state platform, fixture class, and proof limitations.
- Current UI media must follow the root no-people rule and reflect the shipped interface.
- Prefer links to canonical schemas/code over duplicated field inventories that will drift.
- Run the documentation/link gate for changed Markdown.
