# Unsigned Release Artifact Workflow

- Date: 2026-06-11
- Issue: #54 `ci(release): add unsigned release artifact workflow`
- Workflow: `.github/workflows/release.yml`

## Purpose

Maintainers need a way to prove the release matrix can produce artifacts before
release signing, notarization, checksums, and SBOM generation are fully wired.
The release workflow now supports a manual `workflow_dispatch` dry run that
builds unsigned artifacts from a chosen ref and uploads them as GitHub Actions
artifacts instead of publishing them to a GitHub release.

## How To Run

1. Open GitHub Actions.
2. Select `Release: Build & Package App`.
3. Choose `Run workflow`.
4. Enter a branch, tag, or commit SHA in `ref`.
5. Start the workflow and review the uploaded artifacts after the matrix
   completes.

## Behavior

- `release` events keep the existing release-created behavior and pass a
  `release-id` into the reusable build workflow.
- Manual `workflow_dispatch` runs pass an empty `release-id`, so release upload
  steps do not publish to a GitHub release.
- Manual runs keep `upload-artifacts: true`, so artifacts are retained on the
  workflow run for inspection.
- The matrix is still the inherited RapidRAW release matrix. Narrowing or
  expanding platform coverage remains tracked by #52.

## Current Limitations

- Artifacts are unsigned unless platform-specific signing already exists.
- macOS artifacts are not notarized or stapled.
- Checksums and the source dependency SBOM are generated by #55, with details in
  `docs/release/evidence/release-metadata-checksums-sbom-2026-06-11.md`.
- Manual dry runs do not make shippability claims.
- Failure artifact collection is still tracked by #53.

## Validation Expectations

- Workflow syntax must pass `bun run check:actions`.
- PR CI should require the macOS smoke because this changes release workflow
  behavior.
- A future manual dry run should record artifact names, platform rows, failures,
  and skipped signing/notarization status in the PR or issue evidence ledger.
